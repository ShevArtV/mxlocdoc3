mxLocDoc Changelog
==================

2.0.1-pl
--------

- Добавлено: якоря в ссылках. `[текст](./doc.md#раздел)` открывает документ и прокручивает панель к нужному разделу, `[текст](#раздел)` прокручивает текущий документ, не трогая адрес страницы и не сдвигая саму админку. Слаг якоря и id заголовка считаются одинаково, id проставляются заголовкам с первого по шестой уровень (в оглавление справа по-прежнему идут только первые три), а неизвестный якорь просто открывает документ с начала.
- Добавлено в документацию: разделы про якоря и про общий каталог картинок в корне документации (`![x](/assets/file.png)`) — в руководстве по синтаксису Markdown, ru и en.
- Исправлено: в просмотре документа не было видно маркеров у маркированных списков и номеров у нумерованных. ExtJS-ресет админки (`ext-all-notheme-min.css`) гасит `list-style` глобально, поэтому вьюер теперь возвращает маркеры явно — вместе с отступами пунктов и вложенных списков.

2.0.0-pl
--------

- Порт mxLocDoc на MODX Revolution 3. Релиз рассчитан только на MODX 3; линия под MODX 2 остаётся на 1.0.0-pl.
- Сервисный слой переведён на PSR-4 в пространстве имён `MxLocDoc\` с автозагрузкой composer и `bootstrap.php`, который регистрирует сервис `mxlocdoc` в DI-контейнере MODX 3 (`$modx->services`).
- Процессоры переведены на `MODX\Revolution\Processors\Processor`, контроллер менеджера остался на `MODX\Revolution\modExtraManagerController`.
- Инициализация через `$modx->getService()` заменена на контейнер `$modx->services` в коннекторе, контроллере, процессорах и плагине очистки кэша.
- Parsedown подключается через composer (`erusev/parsedown`) вместо вложенной копии.
- Сборка переведена с самописного `modxbuilder` для MODX 2 на сборщик пакетов `modxapp` для MODX 3.
- Требуются PHP >= 8.1 и MODX >= 3.0.0.

1.0.0-pl
--------

- Усилен кэш поиска: подготовленный индекс хранится защищённым PHP-файлом (`<?php return ...;`) вместо сырых сериализованных данных, поэтому открытый наружу каталог кэша не отдаст текст документации как обычный вывод.
- Front matter `hidden: true` теперь учитывается и для страниц, перечисленных в manifest навигации, — так же, как в fallback по файловой системе.
- Исправлены смещения фрагментов в поиске для многобайтового (UTF-8) текста: позиции и длины считаются с учётом кодировки.
- Многоязычный режим объявляется явно новой системной настройкой `mxlocdoc.languages` (коды языков через запятую, по умолчанию `en,ru`); пустое значение отключает автоопределение, и обычная двухбуквенная папка больше не перехватывает дерево документации.

0.1.0-pl
--------

- Первый релиз пакета под MODX 2.
- Просмотр документации в менеджере: три колонки, хлебные крошки, сворачиваемая навигация и оглавление страницы.
- Системные настройки, лексиконы настроек и нормализация опций во время выполнения.
- Поддержка `[[+corePath]]` и путей относительно core для `mxlocdoc.docs_path`.
- Безопасные сервисы файловой системы, процессоры документов и ассетов.
- Построитель навигации с поддержкой manifest `_sidebar.json`/`mxlocdoc.json` и fallback по файловой системе.
- Безопасный рендер Markdown с защищёнными адресами относительных ассетов.
- Живой поиск по Markdown с кэшированием индекса файловой системы в кэше MODX.
- Плагин очистки кэша менеджера для сброса поискового кэша mxLocDoc.
- Многоязычные корни документации и переключатель языка в менеджере.
- Публикуемая документация на английском и русском.
- Лицензия пакета — GNU GPL v2.
